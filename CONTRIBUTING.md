# Contribuindo

1. Crie uma branch `feature/<tema>` a partir de `develop`.
2. Atualize contrato e dicionário quando alterar schema ou regra.
3. Execute as validações locais disponíveis.
4. Abra PR com evidência de teste.
5. Solicite revisão de outro integrante.
6. Faça merge apenas com o pipeline integrado validado.
